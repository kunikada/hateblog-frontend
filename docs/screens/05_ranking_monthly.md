# 月次ランキング

## 画面概要
- **画面名**: 月次ランキング
- **URL**: `/rankings/:year/:month`
- **URLパラメータ**:
  - `year`: 対象年（YYYY形式）
  - `month`: 対象月（MM形式、1-12）
  - `limit`: 取得件数（任意、デフォルト: 100）
- **目的**: 指定年月のブックマーク件数上位エントリーTOP100を表示

### URL例
- `/rankings/2025/01`

## レイアウト構成

### パンくずリスト
- ランキング > 2025年 > 1月

### ページヘッダー
- タイトル: "2025年1月のランキング TOP100"
- 年月ナビゲーション（前月 / 年月ドロップダウン / 翌月）

### ヒーローセクション（任意）
- 月間トップ3を強調（構成は年次と同様）

### 週次ランキングリンク
- 第1〜第4(5)週へのクイックリンク（PC: 水平 / SP: チップ横スクロール）

### メインコンテンツ
- 4位〜100位のランキングリスト（年次と同構成）
- ページネーション（任意）

## ランキングカード構成
- 表示要素・レイアウトは [docs/screens/04_ranking_yearly.md](docs/screens/04_ranking_yearly.md) と同一

## API連携

### 初期データ取得
```
GET /api/v1/rankings/monthly?year={year}&month={month}&limit={number}
```

#### リクエストパラメータ
- `year`: YYYY（必須）
- `month`: MM（1-12、必須）
- `limit`: 取得件数（デフォルト: 100、最大: 100）

#### レスポンス
- `openapi.yaml` を参照

### クリック計測
- エントリーカードは [docs/screens/03_entries_new.md](docs/screens/03_entries_new.md) と同一仕様

## インタラクション

### 年月選択
1. 前月/翌月ボタンで `year, month` を調整して再取得（年またぎ対応）
2. 年月ドロップダウンで任意年月へ遷移

### 週次ランキング遷移
- `/rankings/{year}/week/{week}` へ遷移

### エントリーカード/共有
- クリック計測・共有は [docs/screens/03_entries_new.md](docs/screens/03_entries_new.md) と同一

## 状態管理

### ローカルステート
- `year`, `month`: 対象年月
- `rankings`: `{ rank, entry }[]`
- `total`: 総件数
- `isLoading`: ローディング
- `error`: エラー

### グローバルステート
- なし

## デザインガイドライン
- 基本は [docs/screens/04_ranking_yearly.md](docs/screens/04_ranking_yearly.md) を踏襲
- ヒーロー/順位バッジ/カード寸法は年次と同一でOK

## アクセシビリティ
- `<h1>` でタイトル、`<ol>` + `<li>` でランキング
- 週次リンクは `<nav aria-label="週次ランキング">`

## パフォーマンス
- 初期表示3秒以内
- 同年月データはキャッシュ再利用

## エラーハンドリング
- 400: "年月の形式が正しくありません"
- 404: "該当月のランキングデータが見つかりません"
- 500: "サーバーエラーが発生しました"
- トースト表示 + 再試行ボタン
